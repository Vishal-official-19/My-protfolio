📧 CONTACT FORM BACKEND SETUP GUIDE
=====================================

Your form is now using a complete Node.js backend that sends emails directly to your Gmail! 🎉

✅ WHAT'S INSTALLED:
- backend.js (Email server + File serving)
- Updated form.js (sends to /api/contact endpoint)
- Updated contact.html (all form fields: name, email, phone, address, budget, service, message)

---

⚙️ 3 SIMPLE SETUP STEPS:

STEP 1: Install Nodemailer
──────────────────────────
Run this command in terminal (in your portfolio folder):

  npm install nodemailer

(Or if npm not installed, download Node.js from nodejs.org)


STEP 2: Enable Gmail App Password
──────────────────────────────────
1. Go to: https://myaccount.google.com/apppasswords
2. Sign in with your Gmail account
3. Select "Mail" → Select "Windows Computer" (or device)
4. Copy the 16-character password shown
5. Keep it safe!


STEP 3: Update backend.js with Gmail Details
──────────────────────────────────────────
Find these lines in backend.js (around line 14-15):

  const GMAIL_USER = 'Vishal.studio.live@gmail.com';  // YOUR EMAIL
  const GMAIL_PASSWORD = 'YOUR_GMAIL_APP_PASSWORD';   // PASTE 16-CHAR PASSWORD HERE
  const RECEIVE_EMAIL = 'Vishal.studio.live@gmail.com'; // WHERE EMAILS COME

Update:
  - GMAIL_USER: Your Gmail address (where you enabled app password)
  - GMAIL_PASSWORD: The 16-character password from step 2
  - RECEIVE_EMAIL: Where you want to receive contact form messages

---

🚀 START YOUR SERVER
────────────────────
Open terminal and run:

  npm start

You should see:

  1️⃣ Form submission with all client details
  2️⃣ Confirmation email sent to the client

⚠️ TROUBLESHOOTING
──────────────────

Q: "Failed to send message" error?
A: 
  1. Check GMAIL_USER and GMAIL_PASSWORD are correct in backend.js
  2. Make sure app password (not regular password) is used
  3. Restart server after updating credentials

Q: "Module not found: nodemailer"?
A: Run: npm install nodemailer

Q: Still not working?
A: Check backend.js console for error details
